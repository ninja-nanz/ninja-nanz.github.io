Position: Static, Relative, Absolute, Fixed





"Position" is a CSS property that helps you place an element where you want it to be on a webpage. The different types of position css attributes dicussed below allow for vertical and/or horizontal shifting (top, right, bottom, left) of elements such as divs. This is what it looks like in your css file:

position: relative;

Here are the variations in the different types positioning:




1. Static

This is the default positioning of elements following the "natural flow" rules of the HTML document. 1

position: static;


2. Relative

Relative positioning allows you to move an element in relation to it's original "static" position. For example, the css below will move it 20px above, away from where it was initially.

position: relative; top: 20px

"Relative" positions the element in the normal flow of the document.The other elements dont take up the position of the positioned element ie they behave as if the box has not moved.



3. Absolute

Similar to relative, "absolute" positions the element relative to the html document OR the nearested parent element, such as a container/div. If the parent div is absolute as well, then the element will be relative to that. For example, the css below will move the element 20px below the top of the HTML page.

position: absolute; bottom: 20px

"Absolute" takes the element out of the html document flow and positions it relative to entire document, so other elements behave as if the the positioned element never existed.


4. Fixed

"Fixed" position takes the element out of the flow of the document entirely. This is relative to the viewport (edges) of the browser. For example, the css below will position the element 20px left of the edges of your browser.

position: fixed; left: 20px

The other elements of the page move to fill the element that is fixed, similar to "absolute" position. It is often used for sticky headers and footers and nav bars.


Note: position: inherit   adopts the positioning of it's parent element.
